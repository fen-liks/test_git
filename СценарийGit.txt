
#_Сценарий участия в акции_

**Описание: сценарий описывает участие пользователя в акциях**

**Акторы: сайт (система), пользователь**

**Триггер: пользователь хочет купить пиццу по акции** 

##Основной сценарий:

1. Пользователь выбирает пиццу которая идет по акции

2. Система предлагает добавить топпинг к выбранной пицце

3. Пользователь выбирает нужный топпинг 

4. Система добавляет выбранную пиццу и топпинг в заказ

5. Пользователь проверяет заказ на соответствие выбору 

6. Система запрашивает подтверждение заказа

7. Пользователь подтверждает заказ

8. Система передает заказ менеджеру

##Альтернативный сценарий:

1. 5.1 Пользователь проверяет заказ на соответствие и меняет топпинг

2. Система удаляет старый выбор и меняет на новый

3. Система запрашивает подтверждение заказа

4. Пользователь подтверждает заказ

5. Система передает заказ менеджеру

##Исключения:

1. Пользователь не выбирает ни один из топпингов и заказывает без них

2. Акция закончилась и сайт не может дать подтверждение выбранному заказу: нет больше пицц по акции
